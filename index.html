Question Answer
1. Different types of cables used in networking and connectors: ○ Twisted Pair Cable (Cat5e, Cat6, etc.): Connector - RJ45
○ Coaxial Cable: Connector - BNC, F-type
○ Fiber Optic Cable: Connectors - LC, SC, ST
2. Maximum segment size: ○ Twisted Pair: 100 meters (328 feet) for Cat5e, Cat6.
○ Fiber Optic: Depends on the type; for Single-mode, it can range up to 40 km or more, while Multimode fiber is typically around 2 km.
3. Wires and twists in UTP (Unshielded Twisted Pair): ○ UTP typically consists of 8 wires grouped into 4 pairs. Each pair is twisted to reduce interference.
4. Cable used in LAN: ○ Twisted Pair (Cat5e, Cat6) is commonly used for Ethernet LAN.
5. Use of firewall: ○ A firewall controls incoming and outgoing network traffic by filtering data packets based on a set of security rules, acting as a barrier between secure internal networks and untrusted external ones.
6. Different topologies, their advantages and disadvantages: ○ Star Topology:
■ Adv: Easy to install and manage, if one cable fails it doesn’t affect the others.
■ Disadv: If the central hub fails, the entire network goes down.
○ Bus Topology:
■ Adv: Simple to set up and requires less cable.
■ Disadv: If the main cable fails, the whole network goes down; difficult to troubleshoot.
○ Ring Topology:
■ Adv: Data travels in one direction, reducing the chance of packet collisions.
■ Disadv: A failure in any cable can bring down the entire network.
○ Mesh Topology:
■ Adv: Offers the highest fault tolerance and redundancy.
■ Disadv: Expensive and complex to install due to a large amount of cabling.
**Preferred Topology**: Star topology is generally preferred due to ease of troubleshooting and managing network nodes.
7. IEEE standards for Ethernet LAN: ○ 802.3 for Ethernet.
○ 802.3u for Fast Ethernet.
○ 802.3ab for Gigabit Ethernet over UTP.
○ 802.3z for Gigabit Ethernet over fiber.
8. Considerations when designing a network for 10 PCs: ○ Type of cable (e.g., Cat6), switches, routers.
○ IP addressing scheme (static or dynamic).
○ Network security (firewalls, encryption).
○ Future scalability.
9. 7 Layers of OSI model: ○ Physical
○ Data Link
○ Network
○ Transport
○ Session
○ Presentation
○ Application
10. Working of each layer: ○ Physical Layer: Deals with hardware transmission technologies (e.g., cables, switches).
○ Data Link Layer: Provides node-to-node data transfer (e.g., MAC addressing, Ethernet).
○ Network Layer: Manages data routing, packet forwarding (e.g., IP address).
○ Transport Layer: Ensures reliable data transmission (e.g., TCP, UDP).
○ Session Layer: Manages sessions between applications.
○ Presentation Layer: Handles data formatting, encryption.
○ Application Layer: Provides services to end users (e.g., HTTP, FTP).
11. OSI and TCP/IP reference model diagrams: ○ OSI: 7 layers (mentioned above).
○ TCP/IP: 4 layers (Network Access, Internet, Transport, Application).
12. Difference between OSI and TCP/IP: ○ OSI is a conceptual model with 7 layers, while TCP/IP is a practical implementation with 4 layers.
○ OSI is protocol-independent, whereas TCP/IP is based on a specific protocol suite.
13. Protocols at each OSI layer: ○ Application: HTTP, FTP, DNS
○ Presentation: SSL, TLS
○ Session: RPC, NetBIOS
○ Transport: TCP, UDP
○ Network: IP, ICMP
○ Data Link: Ethernet, PPP
○ Physical: Ethernet cables, wireless standards.
14. Networking devices in TCP/IP layers: ○ Network Access: Switches, Hubs, NICs
○ Internet: Routers
○ Transport: Firewalls
○ Application: Proxy Servers, Web Servers
15. ATM (Asynchronous Transfer Mode): ○ A high-speed networking standard that supports data, voice, and video transmission over the same network.
16. Tunneling: ○ A process where one protocol is encapsulated within another protocol to enable secure data transmission across public networks.
17. Fragmentation: ○ The process of breaking a packet into smaller pieces to fit the maximum transmission unit (MTU) of the network.
20. Two types of transmission technology Broadcast (one-to-many) and Point-to-Point (one-to-one).
21. Subnetting A technique to divide a network into smaller sub-networks to efficiently use IP addresses and improve security.
22. Difference between communication and transmission Communication refers to the process of exchanging data, while transmission refers to the act of sending data from one place to another.
23. Router A device that forwards data packets between networks, directing traffic based on IP addresses.
24. Point-to-Point Protocol (PPP) A data link protocol used to establish a direct connection between two nodes.
25. MAC Address A 48-bit unique identifier for network interfaces, usually written in hexadecimal format (e.g., 00:1A:2B:3C:4D:5E).
26. IP address layer and size IP addresses work at the Network layer of the OSI model. IPv4 is 32 bits; IPv6 is 128 bits.

Class B: 128.0.0.0 to 191.255.255.255
Class C: 192.0.0.0 to 223.255.255.255

Class B: 16,384 networks, ~65,000 hosts
Class C: 2 million networks, 254 hosts.
29. Default subnet masks Class A: 255.0.0.0
Class B: 255.255.0.0
Class C: 255.255.255.0
30. Physical and Logical Address Physical Address refers to MAC address.
Logical Address refers to IP address.
31. Types of transmission media Guided (Twisted Pair, Fiber Optic)
Unguided (Wireless)
32-36. Repeaters, Bridges, Routers, Gateways, Switches Repeater: Amplifies signals to extend the transmission distance.
Bridge: Connects two LANs at the data link layer.
Router: Routes data between networks.
Gateway: Translates protocols between networks.
Switch: Directs data within a network (Layer 2 or Layer 3).
37-40. Hub vs Switch, Switch vs Router, Cutoff switch, Manageable and Unmanageable Switch Hub: Broadcasts data to all devices.
Switch: Directs data to specific devices.
Router: Forwards data across networks.
Cutoff switch: Stops power flow to prevent overload.
Manageable switch: Can be configured; Unmanageable: Basic functionality.
41. Web server Hosts websites and serves web pages to users.
42. Broadcast, Multicast, Unicast Broadcast: One-to-all.
Multicast: One-to-many.
Unicast: One-to-one.
43. FDDI (Fiber Distributed Data Interface) A standard for transmitting data over fiber optic cables in a ring topology.
44. Token Ring/Bus Token Ring: Nodes take turns to send data using a token.
Token Bus: Similar to Token Ring but uses a bus topology.
45. IEEE standards from 802.2 to 802.16 802.2: Logical Link Control.
802.11: Wi-Fi.

46. What is the IP address of our server? The IP address of a server depends on the network setup. You can check the server's IP address using commands like ipconfig (Windows) or ifconfig (Linux) or ask the IT department for the assigned static IP.
47. What are the two lines used in our organization for internet connection? Give 3 points where the difference is there. DSL (Digital Subscriber Line) and Fiber Optic connections, or Leased Line and Broadband connections.
Differences:
1. Speed: Fiber offers higher speeds.
2. Reliability: Leased Line is more stable.
3. Cost: Leased Line is more expensive.
48. What is the difference between leased line and dial-up connection? Leased Line: A dedicated, private line with consistent high speeds.
Dial-up: Uses telephone lines with slower, temporary connections (up to 56 kbps).
Key Differences:
1. Speed: Leased lines offer high speeds, dial-up is slow.
2. Connection: Leased lines are always-on, dial-up is temporary.
3. Cost: Leased lines are more expensive than dial-up.
49. What is meant by ISP? ISP (Internet Service Provider): A company that provides internet access to individuals or organizations (e.g., Comcast, AT&T).
50. What is ICMP? ICMP (Internet Control Message Protocol): A protocol used to send error messages and network diagnostics (e.g., ping).
51. What are the data units at different layers of the TCP/IP protocol suite? 1. Application Layer: Data
2. Transport Layer: Segment (TCP) or Datagram (UDP)
3. Network Layer: Packet
4. Data Link Layer: Frame
5. Physical Layer: Bits
52. What is the difference between ARP and RARP? ARP: Translates an IP address into a MAC address.
RARP: Translates a MAC address into an IP address.
53. What is the minimum and maximum length of the header in the TCP segment and IP datagram? TCP Header: Minimum: 20 bytes, Maximum: 60 bytes.
IP Header: Minimum: 20 bytes, Maximum: 60 bytes.
54. What is the difference between TFTP and FTP application layer protocols? TFTP: A simpler, faster protocol without authentication, uses UDP.
FTP: A more robust protocol with authentication, uses TCP.
55. What are the major types of networks? LAN: Local Area Network (e.g., office network).
WAN: Wide Area Network (e.g., internet).
MAN: Metropolitan Area Network (e.g., citywide network).
PAN: Personal Area Network (e.g., Bluetooth).
56. What are the important topologies for networks? 1. Bus Topology
2. Star Topology
3. Ring Topology
4. Mesh Topology
57. What is mesh network? A network where each node is connected to multiple other nodes, providing redundancy and fault tolerance.
58. Why should you care about the OSI Reference Model? The OSI model standardizes networking protocols, aiding in troubleshooting and understanding communication layers.
59. What is VPN? A secure, encrypted connection over a public network, often used for remote access or privacy.
60. What is Virtual LAN? A logical segmentation of a network to isolate or manage traffic within a physical network.
61. What is packet filter? A packet filter is a network security mechanism that controls data flow based on IP addresses, protocols, and ports, blocking or allowing data packets based on predefined rules.
62. What is traffic shaping? Traffic shaping is a technique used to control the amount of data sent over a network at a time to ensure efficient network performance and avoid congestion.
63. What is meant by flow control? Flow control regulates the amount of data sent between sender and receiver to ensure the receiver doesn’t get overwhelmed with too much data.
64. What is meant by congestion control? Congestion control ensures that network resources aren’t overloaded by limiting the amount of data being transmitted to avoid packet loss and delays.
65. What is IGP (Interior Gateway Protocol)? IGP refers to protocols used within a single autonomous system for routing, like OSPF and RIP.
66. What is EGP (Exterior Gateway Protocol)? EGP refers to protocols used between different autonomous systems, such as BGP (Border Gateway Protocol).
67. What is an autonomous system? An autonomous system (AS) is a collection of IP networks under the control of a single organization, with its own routing policy.
68. What is BGP (Border Gateway Protocol)? BGP is the protocol used for routing data between different autonomous systems on the internet.
69. What is OSPF? OSPF (Open Shortest Path First) is an IGP protocol used within an autonomous system to find the shortest path for data to travel.
70. What is RIP (Routing Information Protocol)? RIP is a distance-vector routing protocol that determines the best route for data based on hop count.
71. What is PPP? PPP (Point-to-Point Protocol) is used to establish a direct connection between two network nodes, often used in dial-up connections.
72. What is DHCP (Dynamic Host Configuration Protocol)? DHCP automatically assigns IP addresses and other network configurations to devices on a network.
73. What is SNMP (Simple Network Management Protocol)? SNMP is used for managing and monitoring network devices like routers, switches, and servers.
74. How are current IPv4 addresses allocated? IPv4 addresses are allocated by regional internet registries (RIRs) based on the geographic location of the organization.
75. How do IPv6 addresses differ from addresses used in current IP version? IPv6 uses 128-bit addresses, which provides a vastly larger address space than the 32-bit IPv4 addresses.
76. How does IPv6 address allocation differ from address allocation used with IPv4? IPv6 uses a hierarchical structure to allocate addresses globally, regionally, and locally. IPv4 uses a mix of classful and classless (CIDR) methods.
77. How is IPv6 supposed to solve address allocation problems with IPv4? IPv6 solves the exhaustion of IPv4 addresses by providing a much larger address pool (128-bit addresses versus 32-bit).
78. How are addresses belonging to different types differentiated from each other? IPv4 addresses are classified into A, B, C, D, and E based on the first few bits. For example, Class A starts with 0, Class B with 10, and so on.
79. What is supernetting and subnetting? Subnetting divides a network into smaller sub-networks for better organization and management. Supernetting combines multiple networks into a larger one, typically to reduce the number of routes in a routing table.

81. What is the network part in the address 172.16.10.50/27? What is the host part? Network part: 172.16.10.32. Host part: 18 (binary host part: 00010010).
82. How many subnets are available in the network mentioned above? How many hosts can be in one subnet? Subnets: 8. Hosts per subnet: 30 (25 - 2).
83. What does the notation 211.22.23.0,3 mean? What addresses belong to this definition? This notation means the first 3 bits are used for subnetting, referring to a range of subnets under the base address 211.22.23.0. The exact range depends on further context like the netmask.
84. What are networking commands? Explain each one? Ipconfig: Displays the IP configuration of a computer (Windows). Traceroute: Shows the route packets take to reach a destination. Ping: Tests connectivity to a specific IP address. Nslookup: Looks up the DNS server for a specific domain.
85. When you give ping command, what is the output after that and also tell meaning of each term? The output typically shows: Reply from: Indicates the response from the destination. TTL (Time to Live): Number of hops the packet can make before being discarded. Time: The round-trip time for the packet. Loss: The percentage of packets lost during the test.
86. What is TFTP and how does it differ from FTP? TFTP (Trivial File Transfer Protocol) is a simple protocol used for transferring files. Unlike FTP (File Transfer Protocol), TFTP does not provide authentication, encryption, or complex command sets. It uses UDP instead of TCP, making it faster but less reliable.
87. What are the main differences between TCP and UDP? TCP (Transmission Control Protocol) is connection-oriented, ensuring reliable communication with error-checking and flow control. UDP (User Datagram Protocol) is connectionless, providing faster transmission without guarantees of delivery or order.
88. What is NAT and how does it work? NAT (Network Address Translation) is a technique used to modify network address information in packet headers while in transit. It allows multiple devices on a local network to share a single public IP address, improving security and conserving IP addresses.
89. What is the purpose of a subnet mask? A subnet mask is used to divide an IP address into network and host portions. It helps routers and switches determine whether a destination IP address is on the same subnet or requires routing.
90. What is DHCP and what role does it play in networking? DHCP (Dynamic Host Configuration Protocol) automates the process of assigning IP addresses and other network configuration parameters to devices on a network, allowing them to communicate effectively.
91. What is a VPN and how does it work? A VPN (Virtual Private Network) creates a secure, encrypted connection over a less secure network, allowing users to send and receive data as if their devices were directly connected to a private network.
92. What are the benefits of using a CDN? A CDN (Content Delivery Network) improves website performance and availability by distributing content across multiple servers located around the world, reducing latency and load times.
93. What is an IP address? An IP address is a unique identifier assigned to devices connected to a network, allowing them to communicate with each other. It can be either IPv4 (32-bit) or IPv6 (128-bit).
94. How does SSL/TLS work to secure communications? SSL (Secure Sockets Layer) and TLS (Transport Layer Security) work by establishing a secure, encrypted connection between a client and a server through a process of authentication and key exchange.
95. What is the difference between a switch and a router? A switch connects devices within a local area network (LAN) and forwards data based on MAC addresses. A router connects different networks and forwards data based on IP addresses.
96. What is a firewall and how does it work? A firewall is a network security device that monitors and controls incoming and outgoing traffic based on predetermined security rules. It acts as a barrier between trusted and untrusted networks.
97. What is the OSI model? The OSI (Open Systems Interconnection) model is a conceptual framework used to understand and implement networking protocols. It consists of seven layers: Physical, Data Link, Network, Transport, Session, Presentation, and Application.
98. What is the difference between HTTP and HTTPS? HTTP (Hypertext Transfer Protocol) is the standard protocol for transferring web pages, while HTTPS (HTTP Secure) adds a layer of security by encrypting data between the client and server using SSL/TLS.
99. What is a public key infrastructure (PKI)? PKI is a framework that uses digital certificates and public-private key pairs to secure communications and transactions over networks. It enables users to exchange data securely.
100. What are the different types of malware? Types of malware include viruses, worms, Trojans, ransomware, spyware, and adware, each designed to disrupt, damage, or gain unauthorized access to systems.
101. What is social engineering in cybersecurity? Social engineering is the manipulation of individuals into divulging confidential information by exploiting human psychology rather than technical hacking techniques.
102. What is a man-in-the-middle attack? A man-in-the-middle attack occurs when an attacker intercepts and alters communication between two parties without their knowledge, often to steal information.
103. What is phishing? Phishing is a cyber attack that involves tricking individuals into providing sensitive information, such as passwords or credit card numbers, by impersonating a trustworthy entity.
104. What is a DDoS attack? A DDoS (Distributed Denial of Service) attack aims to overwhelm a target's resources, such as a server or network, by flooding it with excessive traffic, rendering it unavailable.
105. What is data encryption? Data encryption is the process of converting information into a code to prevent unauthorized access. It uses algorithms to ensure that only authorized parties can decrypt and access the original data.
106. What is a digital signature? A digital signature is a cryptographic method used to verify the authenticity and integrity of a message, software, or digital document, providing assurance that it has not been altered.
107. What is a security breach? A security breach is an incident that results in unauthorized access to data, applications, services, or devices, potentially leading to data loss or compromise.
108. What is risk management in cybersecurity? Risk management in cybersecurity involves identifying, assessing, and mitigating risks to an organization’s information assets to protect against potential threats.
109. What is an intrusion detection system (IDS)? An IDS is a device or software application that monitors network traffic for suspicious activity and alerts administrators of potential security breaches.
110. What is an intrusion prevention system (IPS)? An IPS is a network security device that monitors traffic and can take immediate action to block or prevent malicious activities based on predefined rules.
111. What is the role of a cybersecurity analyst? A cybersecurity analyst is responsible for protecting an organization’s computer systems and networks by monitoring, detecting, and responding to security incidents.
112. What is a threat model? A threat model is a process used to identify potential threats to a system, assess the risks associated with those threats, and determine how to mitigate them.
113. What is a zero-day vulnerability? A zero-day vulnerability is a security flaw in software that is unknown to the vendor and has not yet been patched, making it particularly dangerous until a fix is released.
114. What is a security policy? A security policy is a formal document that outlines an organization’s approach to managing its information security, including rules and procedures for protecting data.
115. What is encryption at rest? Encryption at rest refers to the protection of data stored on physical media, ensuring that it is inaccessible without the proper keys or credentials, even if the storage device is compromised.
116. What is encryption in transit? Encryption in transit refers to the protection of data being transmitted over a network, ensuring that it cannot be intercepted or accessed by unauthorized parties during transmission.
117. What is multi-factor authentication (MFA)? Multi-factor authentication (MFA) is a security mechanism that requires two or more forms of verification before granting access to a system or application, enhancing security beyond just a password.
118. What is a cybersecurity framework? A cybersecurity framework is a set of guidelines and best practices that help organizations manage and reduce cybersecurity risk, often tailored to specific industries or regulatory requirements.
119. What is a cyber threat intelligence (CTI)? Cyber threat intelligence (CTI) is information that helps organizations understand potential cyber threats and vulnerabilities, enabling them to make informed decisions about cybersecurity strategies.
120. What is the difference between symmetric and asymmetric encryption? Symmetric encryption uses the same key for both encryption and decryption, while asymmetric encryption uses a pair of keys (public and private) for these processes, providing enhanced security.
121. What is the principle of least privilege? The principle of least privilege is a security concept that recommends granting users only the access necessary to perform their job functions, minimizing potential security risks.
122. What is a botnet? A botnet is a network of compromised computers or devices controlled by a malicious actor, often used to perform coordinated attacks or distribute malware.
123. What is ransomware? Ransomware is a type of malware that encrypts a victim's files, demanding payment (ransom) for the decryption key, effectively holding the data hostage.
124. What is a honeypot? A honeypot is a security resource that appears to be a vulnerable system, designed to attract and deceive attackers, helping to study their methods and improve security.
125. What is a security incident response plan (SIRP)? A SIRP is a documented approach outlining how an organization will respond to and manage a security incident, including detection, containment, eradication, and recovery processes.
126. What is penetration testing? Penetration testing is an authorized simulated cyber attack on a system or network to identify vulnerabilities and assess the effectiveness of security measures.
127. What is a digital forensics investigation? A digital forensics investigation involves the collection, preservation, analysis, and presentation of electronic evidence to support legal proceedings or internal investigations.
128. What is an access control list (ACL)? An access control list (ACL) is a set of rules that defines who or what is allowed to access resources within a computing environment, such as files, directories, or networks.
129. What is a security audit? A security audit is a systematic evaluation of an organization's information systems and practices to ensure compliance with security policies and identify potential vulnerabilities.
130. What is an exploit? An exploit is a piece of software or code that takes advantage of a vulnerability in a system or application to perform unauthorized actions.
131. What is a vulnerability assessment? A vulnerability assessment is a systematic evaluation of a system or application to identify security weaknesses and recommend measures to mitigate them.
132. What is a secure coding practice? Secure coding practices are guidelines and techniques aimed at preventing security vulnerabilities during the software development process, ensuring the creation of robust and secure applications.
133. What is the role of a chief information security officer (CISO)? A CISO is an executive responsible for establishing and maintaining an organization's information security strategy, policies, and procedures to protect its information assets.
134. What is an incident report? An incident report is a documented account of a security incident, detailing the nature of the incident, the response actions taken, and recommendations for future prevention.
135. What is a security operations center (SOC)? A security operations center (SOC) is a centralized unit that monitors, detects, responds to, and mitigates security incidents in real time to protect an organization’s information systems.
136. What is a threat vector? A threat vector is a path or method used by cybercriminals to exploit vulnerabilities and gain unauthorized access to a system or network.
137. What is patch management? Patch management is the process of managing updates for software applications and technologies, including applying patches to fix vulnerabilities and improve functionality.
138. What is network segmentation? Network segmentation involves dividing a computer network into smaller, isolated segments to improve performance, security, and management.
139. What is a security breach notification? A security breach notification is a formal communication informing affected parties of a data breach, detailing the nature of the breach and steps being taken to address it.
140. What is a security token? A security token is a physical or digital device used to authenticate a user’s identity and provide secure access to a system or application, often used in two-factor authentication.
141. What is a digital certificate? A digital certificate is an electronic document used to prove the ownership of a public key, containing information about the key, its owner, and the digital signature of an authority.
142. What is an application security testing? Application security testing involves evaluating applications for vulnerabilities and security weaknesses through various methods, such as static analysis, dynamic analysis, and penetration testing.
143. What is a supply chain attack? A supply chain attack targets the vulnerabilities within the supply chain of an organization, compromising third-party vendors or service providers to gain access to the primary target.
144. What is the difference between a vulnerability and a threat? A vulnerability is a weakness in a system or application that can be exploited by threats, while a threat is a potential danger that could exploit a vulnerability to cause harm.
145. What is a security framework? A security framework is a structured set of guidelines and best practices that organizations use to manage and mitigate cybersecurity risks, often aligned with regulatory requirements.
146. What is data loss prevention (DLP)? Data loss prevention (DLP) refers to strategies and tools used to prevent sensitive data from being lost, misused, or accessed by unauthorized users.
147. What is an endpoint security? Endpoint security refers to a strategy focused on protecting devices (endpoints) connected to a network, such as laptops, smartphones, and servers, from security threats.
148. What is a cybersecurity incident? A cybersecurity incident is any event that compromises the confidentiality, integrity, or availability of an organization's information or systems.
149. What is a threat landscape? The threat landscape refers to the evolving environment of potential threats and vulnerabilities that organizations must navigate to protect their information assets.
150. What is security information and event management (SIEM)? SIEM is a security solution that aggregates and analyzes security data from across an organization’s network in real time, providing insights into potential security threats.
151. What is a security incident? A security incident is an event that compromises the integrity, confidentiality, or availability of an organization’s information, often requiring a response.
152. What is a cyber attack? A cyber attack is any attempt to gain unauthorized access to a computer system, network, or data, often with the intent to steal, damage, or disrupt.
153. What is incident response? Incident response is the process of identifying, managing, and mitigating security incidents, ensuring that an organization can recover and prevent future occurrences.
